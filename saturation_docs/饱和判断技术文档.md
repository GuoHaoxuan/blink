## 1. 背景

HXMT（慧眼）卫星高能 X 射线望远镜（HE）在极高计数率事件（如伽马射线暴 GRB）期间，探测器前端电子学的数据缓冲区可能溢出，导致部分物理事例（event）数据丢失。我们将这种状态称为**饱和（saturation）**。

准确识别饱和时间区间对科学分析至关重要：
- 饱和区间内的光变曲线不反映真实源流量
- 需要对饱和区间内的计数率进行死时间校正或标记排除
- 1B 级数据在原始管线中不做饱和标注，需要自行判断

## 2. 硬件架构

### 2.1 数据通路概览

```
NaI/CsI 探测器
    ↓ 模拟信号
前端电子学 (ASIC)
    ↓ 数字事例
FPGA
    ↓ 写入
FIFO A (M67204H, 硬件环形缓冲区)
    ↓ MCU读取
MCU (8051 单片机, PDAUA)
    ↓ 打包写入
FIFO B
    ↓ 1553B总线
星务计算机 → 下传
```

### 2.2 FIFO A — M67204H 芯片

- **类型**：硬件 FIFO，环形缓冲区结构
- **关键特性**：
  - 有 Full 标志位（`FIFOAFull1`, `FIFOAFull2`），由硬件自动置位
  - **满时不覆盖**：当 FIFO 满时，FPGA 端写入被阻塞，新事例直接丢弃
  - MCU 可通过写 `ResetFIFOA = 0x00` 复位整个 FIFO
- **地址映射**：`0xE004`（读）/ `0xE001`（复位）

### 2.3 MCU — 8051 单片机

MCU 是**单线程**处理器，通过主循环轮询处理所有任务。

#### 主循环结构（简化）

```c
while(1) {
    ResetWDI = ~ResetWDI;       // 看门狗喂狗
    SearchStack();               // 处理 1553B 消息
    ReStartTimer();              // GPS 同步
    FIFOAFullReset();            // 检查 FIFO A 是否满，满则复位
    
    if (ReadFlag == 1) {         // 每秒一次
        HandleEnginnerLVDS();    // 工程数据处理 (~2ms)
    }
    
    if (INT1Flag == 1) {         // FPGA 中断触发
        HandlePhysicalLVDS();    // 物理数据处理 (~7ms)
        INT1Flag = 0;
        EX1 = 1;                 // 重新使能中断
    }
    
    Refresh();
}
```

#### 关键时间参数

| 操作 | 耗时 | 说明 |
|------|------|------|
| HandlePhysicalLVDS | ~7ms | 从 FIFO A 读 109 个事例，打包写入 FIFO B |
| HandleEnginnerLVDS | ~2ms | 处理工程遥测数据 |
| FIFOAFullReset | 极短 | 检查 Full 标志，触发复位 |

### 2.4 物理数据处理流程 — HandlePhysicalLVDS

```
1. 复位 FIFO B
2. 写入 CCSDS 包头（6 字节）
3. 循环 109 次：
   a. 搜索 0x5A 起始标记
   b. 从 FIFO A 读取 8 字节事例数据
   c. 直接转写入 FIFO B
4. 写入 UTC 时间尾部（4 字节）
5. 通知 FPGA 开始下一轮 LVDS 传输
```

每个 CCSDS 包 = 6 字节包头 + 872 字节载荷（109 × 8 字节事例）+ 4 字节 UTC 尾 = **882 字节**

## 3. 饱和丢数机制

### 3.1 两种丢数模式

#### 模式一：包内无痕丢数（常见）

```
时间线
─────────────────────────────────────────────
FPGA写入:  ████████████████████████████████████
MCU读取:   ████ HandlePhysicalLVDS (~7ms) ████
FIFO状态:  ░░░░░▓▓▓FULL▓▓▓░░░░░░░░░░░░░░░░░░░
                    ↑ 丢弃    ↑ 恢复
```

**过程**：
1. MCU 正在 `HandlePhysicalLVDS()` 中逐字节读取 FIFO A
2. 高计数率下，FPGA 写入速度 > MCU 读出速度
3. FIFO A 在某一刻被写满，FPGA 无法写入，新事例被丢弃
4. MCU 继续读取，读出一些数据后 FIFO A 不再满，FPGA 恢复写入
5. `HandlePhysicalLVDS()` 完成后回到主循环，`FIFOAFullReset()` 检查时 FIFO A 已经不满 → **不触发复位**

**结果**：
- 丢失少量事例
- CCSDS 包结构完整，包间时间间隔无异常
- **无法通过包间 gap 检测到**

**为什么常见**：
- 只要写速率略高于读速率，FIFO 就会在处理包的 7ms 内短暂达到满状态
- 但因为 MCU 一直在读，很快又恢复了

#### 模式二：整包丢失 / FIFO 复位（较少但影响大）

```
时间线
─────────────────────────────────────────────
FPGA写入:  ██████████████████████████████████████
MCU读取:   ████ Handle ████ FIFOAFullReset() ██
FIFO状态:  ░░░░░▓▓▓▓FULL▓▓▓▓▓▓STILL FULL▓▓▓▓▓▓
                                   ↑ 复位！清空所有数据
```

**过程**：
1. 写入速率**极高**，在 `HandlePhysicalLVDS()` 的 7ms 期间写入量远超读出量
2. MCU 处理完一个包回到主循环时，FIFO A **仍然是满的**
3. `FIFOAFullReset()` 检测到 `FIFOAFull1 && FIFOAFull2 == 0`（Full 标志有效）
4. 执行 `ResetFIFOA = 0x00`，**清空整个 FIFO A**
5. 清除 `INT1Flag`，重新使能外部中断

**结果**：
- 丢失整个 FIFO 缓冲区的数据
- 相邻 CCSDS 包之间出现**大的时间间隔**
- **可以通过包间 gap 检测到**

**为什么影响大**：
- 一次复位丢失大量连续事例
- 在光变曲线上表现为计数率突降或时间间隙

### 3.2 对比总结

| | 模式一：包内无痕丢数 | 模式二：FIFO 复位整包丢失 |
|---|---|---|
| **触发条件** | 写速率 > 读速率，但差距不大 | 写速率 >> 读速率 |
| **丢失量** | 少量事例 | 整个 FIFO 缓冲区 |
| **CCSDS 包影响** | 包结构完整 | 包间出现大 gap |
| **可检测性** | ❌ 无法通过包间 gap 检测 | ✅ 可通过包间 gap 检测 |
| **频率** | 饱和期间常见 | 极高计数率时出现 |
| **影响程度** | 较小 | 显著 |

## 4. 饱和区间检测算法

### 4.1 基本思路

通过检测 CCSDS 包之间的**时间间隔异常**来识别发生了 FIFO 复位（模式二）的时间区间。

### 4.2 算法流程

```
┌─────────────────────────────────┐
│  1. 解析所有 CCSDS 包            │
│     parse_events() → 事例列表    │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│  2. 时间重建                     │
│     reconstruct_with_wrap_tracking() │
│     → 每个包的 MET 时间列表       │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│  3. 计算每个包的时间范围          │
│     PackInfo { min_time, max_time } │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│  4. 按 min_time 排序             │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│  5. 滑窗检测包间 gap             │
│     gap = next.min_time          │
│         - current.max_time       │
│     if gap > threshold → 饱和区间 │
└─────────────┬───────────────────┘
              ↓
┌─────────────────────────────────┐
│  6. 多机箱合并（A/B/C 取并集）   │
│     任一机箱饱和 → 该时段判定饱和 │
└─────────────────────────────────┘
```

### 4.3 Gap 阈值讨论

当前实现使用固定阈值 `GAP_THRESHOLD = 6.9ms`，但这**不够准确**。

正常包间隔取决于实时事例率 R：

$$T_{packet} \approx \frac{109}{R}$$

| 事例率 R (evt/s) | 正常包间隔 | 6.9ms 阈值表现 |
|------------------|-----------|---------------|
| 10,000 | ~10.9ms | ⚠️ 误判：正常间隔已超过阈值 |
| 15,000 | ~7.3ms | ⚠️ 边界情况 |
| 20,000 | ~5.5ms | ✅ 能区分 |
| 50,000 | ~2.2ms | ✅ 灵敏度足够 |

**改进方向**：
- 使用自适应阈值，基于局部事例率动态计算
- 或直接对比相邻包间隔与该时段平均包间隔的偏离程度

### 4.4 时间重建算法

饱和区间检测的前提是准确的事例时间重建。

#### 事例类型

每个 8 字节事例通过 CRC 校验和类型标志位（bit 5:4）区分：

| 类型 | 标志位 | 内容 |
|------|--------|------|
| Event | 0x00 / 0x20 | 物理事例：channel + ptime |
| Second | 0x10 | 秒标记：stime（整秒 MET）+ ptime |
| Error | 其他 | CRC 校验失败 |

#### 重建公式

$$MET = met\_anchor + (wrap\_count \times 524288 + ptime - anchor\_ptime) \times 2\mu s + 4.0s$$

其中：
- `met_anchor = stime + offset`（来自最近的有效 Second 事例）
- `ptime`：19 位精确时间计数器（2μs 分辨率）
- `wrap_count`：ptime 回绕次数（回绕周期 = 2^19 × 2μs ≈ 1.048576s）
- `4.0s`：1B → 1K 经验校正

#### 已知问题

饱和间隙后恢复时，Second 事例的 `ptime` 固定为 0，导致假回绕检测：

```
间隙前: Event ptime=500000
间隙后: Second ptime=0  ← 被误判为回绕
        Event ptime=100  ← 正常递增
```

结果：多计一次回绕 → MET 偏移 +1.048576s

**修复方向**：分离 `anchor_ptime`（用于 MET 计算）和 `chain_ptime`（用于 Event 间的回绕检测），避免 Second 的 `ptime=0` 污染回绕判断。

## 5. 三机箱合并策略

HE 望远镜有三个独立机箱（Box A / B / C），每个机箱有独立的 FIFO A 和 MCU。

判定策略：**任一机箱饱和 → 该时间段判定为饱和**

```rust
// 合并算法
1. 分别对三个机箱扫描饱和区间
2. 收集所有区间，按起始时间排序
3. 合并重叠区间（取并集）
4. 最终得到统一的饱和时间段列表
```

判断某一时刻是否饱和时，使用**二分查找**在已排序的区间列表中定位。

## 8. 静默丢数检测（包内无痕丢数）

### 8.1 原理

模式一（包内无痕丢数）无法通过包间 gap 检测，但可以通过**包内事件间隔统计异常**来发现。

在 FIFO 拥塞期间，MCU 读速率（~15797 evt/s）成为瓶颈，事件以近乎均匀的间隔被读出。如果 FIFO 在读取过程中短暂满溢后恢复，丢弃的事件会在包内产生一个异常大的时间间隔。

### 8.2 检测方法

对每个高事件率包（包内事件率 > MCU 读速率 15797 evt/s）：

1. 计算包内事件间隔序列 `dt[i] = t[i+1] - t[i]`
2. 用过滤后的间隔（< 1ms）估算包内真实事件率 `λ = 1 / mean(dt_filtered)`
3. 对每个间隔计算泊松概率 `log₁₀(p) = -λ × dt / ln(10)`
4. 若 `log₁₀(p) < -10`，标记为静默丢数可疑位置

**关键改进**：lambda 必须用过滤后的间隔估算，否则大 gap 本身会拉低 lambda，导致同包内其他较小的异常间隔漏检。

### 8.3 检测结果

| GRB | Box A | Box B | Box C | 合计 suspects | 独立包数 |
|-----|-------|-------|-------|--------------|---------|
| 200415A | 1 | 0 | 0 | **1** | 1 |
| 221009A | 42 | 42 | 39 | **138** | ~90 |
| 260226A | 0 | 0 | 0 | **0** | 0 |

### 8.4 典型特征

- **位置集中在包头/包尾**：大多数异常间隔出现在 `evt_gap_idx=0`（包头）或 `evt_gap_idx=106`（包尾），对应 MCU 在包边界处开始/结束读取 FIFO 的时刻，此时 FIFO 写入积压最容易发生满溢
- **间隔规模**：典型 500μs ~ 6000μs（正常间隔 ~10-60μs），对应丢失数十到数百个事件
- **与 FIFO 复位的关系**：静默丢数集中在 FIFO 复位（模式二）密集区附近，说明两种模式是同一拥塞过程的不同严重程度表现
- **GRB 强度相关**：221009A（史上最亮 GRB）有大量静默丢数，260226A（较弱）无静默丢数

### 8.5 可视化工具

- `analyze_silent_drop.py`：扫描所有包，检测异常间隔，输出 suspects JSON
- `plot_silent_drop.py`：为每个可疑包画三面板图（概览 ±100ms、放大到包级、包内间隔柱状图），同一包内多个异常间隔合并显示

## 9. 饱和区域光变曲线重建

### 9.1 目标

在 FIFO 复位导致的数据丢失区间（gap）内，估算丢失的事件数量并重建光变曲线的时间轮廓。

### 9.2 FIFO 复位丢数模型

#### 9.2.1 单次 FIFO 复位的数据丢失

每个 FIFO reset gap（相邻两包之间的时间间隙）对应**恰好一次** FIFO 复位。理由：FIFO 复位后清空，重新填满需要 455/R_true 秒（即使 R_true=50000 也需 ~9ms），而主循环一轮只要微秒级，不可能在一轮内再次填满触发复位。

丢失事件数：

$$N_{lost} = R_{true} \times T_{gap}$$

其中：
- $R_{true}$：真实事件率
- $T_{gap}$：gap 持续时间（下一包首事件 - 上一包末事件）

注意：FIFO 中被清空的 ~455 个事件的时间戳落在 gap 时间范围内，已被 $R_{true} \times T_{gap}$ 包含，不应重复计算。

#### 9.2.2 真实率 R_true 的估算

**关键物理约束**：在 FIFO 持续满载时，MCU 每读走一个事件空出一个位置，FPGA 立刻写入当前时刻的新事件。因此 FIFO 中相邻事件的 ptime 间距 ≈ 1/R_read（MCU 读速率），而非 1/R_true。包跨时 T_span ≈ 109/R_read，**不能直接用于估算 R_true**。

但 FIFO 复位后，FIFO 清空，新事件直接进入，不存在满载阻塞。因此 **post-reset 包**（gap 后第一个包）的事件 ptime 间距反映 R_true：

$$R_{true} \approx \frac{109}{T_{span,post-reset}}$$

实测验证（221009A Box A）：pre-reset 包与 post-reset 包的 span 比值均值 ~0.93，说明 R_true 仅比 R_read 高约 10-20%。

#### 9.2.3 三机箱交叉参考

三个 box（A/B/C）有独立的 FIFO 和 MCU，但观测同一 X 射线源，计数率成比例。当一个 box 饱和时，其他 box 可能仍有完好数据。

221009A 统计：

| 情况 | Box A | Box B | Box C |
|------|-------|-------|-------|
| 另外两个 box 都未饱和 | 49.2% | 38.3% | 46.8% |
| 至少一个 box 未饱和 | 92.0% | 92.0% | 92.6% |
| 三个同时饱和 | 8.0% | 8.0% | 7.4% |

92% 的 gap 至少有一个参考 box 可用。

### 9.3 三种饱和区域及重建策略

根据饱和程度不同，分为三种情况，各有对应的检测和重建策略：

#### 9.3.1 轻度饱和 — 静默丢数（模式一）

**判定条件**：邻居包事件率 > MCU 读速率（15000 evt/s）

**物理机制**：FIFO 短暂满溢 → 集中丢几个到几十个事件 → 恢复。丢失是突发性的，表现为包内异常大间隔。

**检测方法**：
1. 对每个高率包，用过滤后间隔（< 1ms）估算 $\lambda$
2. 对拥塞宽包（span > 邻居中位 span × 3），用邻居事件率做 $\lambda$
3. 泊松检验：$\log_{10}(p) = -\lambda \cdot \Delta t / \ln(10) < -10$ → 标记为静默丢数

**重建方法**：
- 总量：$N_{lost} = \lambda \cdot \Delta t - 1$
- 形状：交叉参考其他未饱和 box 的事件分布，用 shape bin (1ms) + 均值填充空 bin
- 参考覆盖率 < 30% 时直接均匀分配

#### 9.3.2 深度饱和 — 包级修正

**判定条件**：邻居包事件率 < MCU 读速率（15000 evt/s），即所有邻居都是拥塞包

**物理机制**：MCU 每 ~8ms 读一次 FIFO，读取 ~109 个事件在 ~0.5ms 内完成（burst），其余 ~7.5ms 为主循环开销期间。FIFO 在此期间持续满载，新事件无法写入被丢弃。

**检测方法**：通过邻居率判定。深度饱和区不做逐间隔的泊松检测（因为 $\lambda$ 无法从数据可靠估算）。

**重建方法**：
- 用包内 burst 部分（间隔 < 1ms）的计数率作为 $R_{true}$
- $N_{fill} = R_{true} \times T_{gap,intra-packet}$（gap = 包内 > 1ms 的大间隔总时长）
- 补全事件均匀分配在包内各个 gap 间隔中

#### 9.3.3 FIFO 复位 — 整包丢失（模式二）

**检测方法**：包间 gap > baseline × 100（GAP_FACTOR），且最大事件率 > 15000。对深度饱和区，用附近包的率代替相邻包的率判断。

**R_true 估算**：
- 非深度饱和：$R_{true} = 109 / T_{span,post-reset}$（post-reset 包的跨时）
- 深度饱和：用邻近包的 burst rate（包内过滤间隔的率）

**重建方法**：
- 总量：$N_{lost} = R_{true} \times T_{gap}$
- 形状函数：汇总所有参考 box 的事件构建 shape bin
  - 跳过参考 box 在不可信区间内的 bin（FIFO reset gap + 拥塞宽包 + 包内异常间隔）
  - 参考覆盖率 < 30% → 均匀分配
  - 参考覆盖率 ≥ 30% → 用有值 bin 的均值填充空 bin
- 归一化到 $N_{lost}$ 后按形状分配事件

### 9.4 不可信区间检测

交叉参考的质量取决于参考 box 的数据是否可靠。以下时间区间被标记为不可信，参考时跳过：

1. **FIFO reset gap**：整包丢失的时间段，无事件
2. **拥塞宽包**：包跨时 > 邻居中位跨时 × 3，事件分布反映 MCU 读取模式而非真实光变
3. **含异常间隔的包**：包内有泊松异常间隔（用邻居率做 $\lambda$ 判定），整个包的事件分布不可信

### 9.5 处理原则

1. **三种重建独立进行**，都只用原始观测事件做交叉参考，不互相影响
2. **总量由物理模型决定**（$R_{true} \times T_{gap}$ 或 $\lambda \times \Delta t - 1$），形状由交叉参考或均匀分配决定
3. **归一化保证总量守恒**：无论形状函数如何，补全事件总数不变

### 9.6 实现

Rust CLI 命令：

```
HXMT_1B_DIR=data/1B ./target/release/blink_cli <datetime> --reconstruct <center_met> <half_window> [bin_width]
```

输出 CSV 格式：`box,bin_center,observed,reconstructed,filled_gap,filled_sd`

- `filled_gap`：FIFO reset gap 补全的事件率
- `filled_sd`：静默丢数 + 深度饱和修正的事件率

核心函数：
- `detect_silent_drops()`：泊松方法检测包内异常间隔（轻度饱和区）
- `reconstruct_silent_drops()`：交叉参考填充静默丢数
- `reconstruct_deep_saturation()`：burst rate 填充深度饱和包内 gap
- `reconstruct_gaps()`：交叉参考填充 FIFO reset gap
- `detect_unreliable_intervals()`：标记不可信时间区间

### 9.7 局限性

1. **包间中等 gap 漏检**：GAP_FACTOR=100 阈值下，10x~50x 的包间 gap 不会被检测为 FIFO reset，在图上表现为窄缝
2. **深度饱和 R_true 精度**：burst rate 反映 MCU 读速率与 FIFO 写入速率的耦合，可能不等于真实源率
3. **三 box 同时饱和**：~8% 的 gap 无参考 box，走均匀分配
4. **FIFO 容量**：455 事件（M67204H 4096 字节）用于一致性检查，不直接影响重建总量
